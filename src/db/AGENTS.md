# AGENTS.md — src/db/

Drizzle ORM over `postgres` (the `postgres` npm package, not `pg`).

## Client setup

`src/db/index.ts` exports the `db` client and re-exports `sql` from drizzle-orm:

```ts
import { drizzle } from "drizzle-orm/postgres-js";
import postgres from "postgres";

export { asc, desc, eq, sql } from "drizzle-orm";

const client = postgres(process.env.DATABASE_URL!);
export const db = drizzle(client, { casing: "snake_case" });
```

**Always import `sql`, `eq`, `asc`, `desc` (and any other drizzle helpers) from `@/db`**, not directly from `drizzle-orm`. This avoids
module resolution conflicts caused by the dual drizzle-orm pnpm instance.

## Query rules

- **Never use `drizzle-orm/relations` or `db.query`** — use explicit `.select().from()` queries only.
- Use `sql<T>` template tag for raw SQL expressions (aggregates, casts, etc.).
- Prefer `cast(... as integer)` / `cast(... as numeric(p,s))` in raw SQL to control return types precisely.
- `casing: "snake_case"` is set globally — TypeScript field names are camelCase, DB columns are snake_case automatically.

## Schema

Tables live in `src/db/schema/`. Re-export everything through `src/db/schema/index.ts`.
Import tables in routers via `@/db/schema` or `@/db/schema/<table>`.

## Migrations

Generated by `drizzle-kit` into `src/db/migrations/`.
- Run `npx drizzle-kit generate` to generate a new migration after schema changes.
- Run `npx drizzle-kit migrate` to apply migrations.
- Never edit migration files manually.

## Environment

`DATABASE_URL` is the only required env var. Format: `postgres://user:pass@host:port/db`.

## Aggregate query pattern

```ts
import { db, sql } from "@/db";
import { roasts } from "@/db/schema";

const [result] = await db
  .select({
    totalRoasts: sql<number>`cast(count(${roasts.id}) as integer)`,
    avgScore: sql<string | null>`cast(avg(${roasts.score}) as numeric(3,1))`,
  })
  .from(roasts);

return {
  totalRoasts: result?.totalRoasts ?? 0,
  avgScore: result?.avgScore ? Number(result.avgScore) : 0,
};
```

Note: `avg()` returns `numeric` from Postgres, which arrives as a string in JS. Always
convert with `Number()` and guard against `null` (empty table).
